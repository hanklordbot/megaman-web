## 2. 系統設計

### 2.1 角色移動系統

| 參數 | 數值 | 說明 |
|------|------|------|
| 移動速度 | 1.5 格/秒 | 水平移動速度 |
| 跳躍高度 | 5 格 | 最大跳躍高度 |
| 重力加速度 | 9.8 格/秒² | 下落加速度 |
| 跳躍時可射擊 | 是 | 空中可發射子彈 |
| 移動時可射擊 | 是 | 行走中可發射子彈 |

**操作定義**:
- 左/右：水平移動
- 跳躍鍵：跳躍（按住時間影響高度）
- 射擊鍵：發射武器
- 選單鍵：開啟武器選擇畫面

## 3. 關卡設計

### 3.1 關卡結構

遊戲共 10 個關卡：6 個 Robot Master 關卡 + 4 個 Dr. Wily 城堡關卡。

**流程**:
1. 玩家從關卡選擇畫面自由選擇 6 個 Robot Master 關卡
2. 完成全部 6 關後解鎖 Dr. Wily 城堡
3. 城堡 4 關為線性順序，含 Boss Rush

### 3.3 Dr. Wily 城堡關卡

| 關卡 | 特色 | Boss |
|------|------|------|
| 城堡 Stage 1 | 高難度平台跳躍 | Yellow Devil（大型 Boss） |
| 城堡 Stage 2 | 複合機關 | Copy Robot（複製 Mega Man） |
| 城堡 Stage 3 | Boss Rush | 重新挑戰 6 位 Robot Master |
| 城堡 Stage 4 | 最終關卡 | Dr. Wily Machine（兩階段） |

### 3.4 關卡設計原則

1. **教學設計**：每關前 2 個畫面引入新機制，無致命危險
2. **難度曲線**：中段提升難度，Boss 前設置補給點
3. **Checkpoint**：每關設 2-3 個重生點
4. **畫面捲動**：水平捲動為主，部分關卡含垂直段
5. **每畫面尺寸**：256x240 像素（NES 標準解析度）

## 5. UI 設計

### 5.1 畫面架構

```
┌─────────────────────────────────────┐
│ [HP BAR]          [SCORE] [LIVES]   │  ← HUD（常駐）
├─────────────────────────────────────┤
│                                     │
│                                     │
│          遊戲畫面區域                │
│          (256 x 224 px)             │
│                                     │
│                                     │
└─────────────────────────────────────┘
```

### 5.2 HUD 元素

| 元素 | 位置 | 說明 |
|------|------|------|
| 玩家 HP 條 | 左上垂直 | 28 格血量條，由上往下減少 |
| Boss HP 條 | 右上垂直 | Boss 戰時顯示，28 格 |
| 武器能量條 | HP 條右側 | 當前特殊武器剩餘彈藥 |
| 得分 | 上方中央 | 6 位數字 |
| 殘機數 | 右上 | 數字顯示 |

### 5.3 畫面流程

```
[開機畫面] → [標題畫面] → [關卡選擇] → [遊戲中] → [Boss 戰]
                              ↓                         ↓
                         [武器選單]              [武器獲得演出]
                              ↓                         ↓
                         [遊戲中]              [回到關卡選擇]
                                                        ↓
                                              [Wily 城堡]（全破後）
                                                        ↓
                                              [結局畫面]
```

### 5.4 各畫面規格

#### 標題畫面
- 遊戲 Logo 居中
- "PRESS START" 閃爍文字
- 背景：簡潔深色

#### 關卡選擇畫面
- 3x2 格子排列 6 位 Robot Master 頭像
- 中央為 Mega Man 頭像（游標起始位置）
- 已通關 Boss 頭像變暗/打勾
- 游標移動有音效回饋

#### 武器選單（暫停畫面）
- 暫停遊戲，全畫面覆蓋
- 列出所有已獲得武器 + 剩餘彈藥
- 游標選擇切換武器
- 顯示 E-Tank 數量（如有）

#### Game Over 畫面
- "GAME OVER" 文字
- 選項：CONTINUE / STAGE SELECT
- CONTINUE：從當前關卡重新開始
- STAGE SELECT：回到關卡選擇畫面

## 7. 技術規格

### 7.1 技術棧

| 項目 | 技術選擇 |
|------|----------|
| 語言 | TypeScript |
| 渲染引擎 | PixiJS v7+ |
| 建置工具 | Vite |
| 套件管理 | npm |
| 目標環境 | 現代瀏覽器（Chrome、Firefox、Safari、Edge） |
| 部署 | 靜態網頁（可部署至 GitHub Pages / Vercel / S3） |

### 7.2 畫面規格

| 項目 | 數值 |
|------|------|
| 遊戲解析度 | 256 x 240 像素（內部邏輯解析度） |
| 顯示縮放 | 整數倍縮放適應視窗（2x~4x） |
| Tile 大小 | 16 x 16 像素 |
| Sprite Sheet 格式 | PNG + JSON atlas（TexturePacker 格式） |
| 幀率 | 60 FPS（requestAnimationFrame） |
| 像素渲染 | `PIXI.SCALE_MODES.NEAREST`（保持像素銳利） |

### 7.3 專案架構

```

├── src/
│   ├── main.ts              # 進入點
│   ├── game/
│   │   ├── Game.ts          # 主遊戲迴圈
│   │   ├── scenes/          # 場景管理（標題、選關、遊戲中、結局）
│   │   ├── entities/        # 角色、敵人、子彈
│   │   ├── systems/         # 物理、碰撞、武器、AI
│   │   ├── levels/          # 關卡資料（Tiled JSON）
│   │   └── ui/              # HUD、選單
│   ├── assets/              # 美術、音效資源

├── tsconfig.json
└── vite.config.ts
```

### 7.4 物理系統
- 格子基礎碰撞（Tile-based collision）
- AABB 碰撞檢測（使用自製輕量物理，不依賴外部物理引擎）
- 單向平台支援（可從下方跳上）
- 固定時間步長（Fixed timestep: 1/60 秒）

### 7.5 攝影機
- 水平捲動：跟隨玩家，玩家保持在畫面 1/3 處
- 垂直段：鎖定垂直捲動
- 房間切換：整個畫面滑動過渡（0.5 秒）
- 實作方式：PixiJS Container 位移

### 7.6 輸入系統
| 輸入方式 | 支援 |
|----------|------|
| 鍵盤 | 方向鍵 + Z（射擊）+ X（跳躍）+ Enter（暫停） |
| 手把 | Gamepad API 支援（選配） |
| 觸控 | 虛擬按鍵（行動裝置，選配） |

### 7.7 音效系統
- 使用 Web Audio API（透過 PixiJS Sound 或 Howler.js）
- BGM 格式：OGG + MP3（雙格式相容）
- SE 格式：WAV（低延遲）
- 8-bit chiptune 風格音源

### 7.8 關卡編輯
- 使用 Tiled Map Editor 製作關卡
- 匯出 JSON 格式，遊戲端解析載入
- 支援多圖層：背景層、碰撞層、物件層（敵人/道具生成點）

### 7.9 效能目標
| 指標 | 目標 |
